# 记忆生命周期管理

## 记忆的产生

```
对话事件 → 判断重要性 → 写入对应层级
```

### 写入规则

| 触发条件 | 写入位置 | 示例 |
|----------|----------|------|
| 重大决策 | `MEMORY.md` | "默认路由改为 us-tianqi" |
| 日常事件 | `memory/YYYY-MM-DD.md` | "完成了 5 个 ASIN 导出" |
| 长任务进度 | `memory/hot/HOT_MEMORY.md` | "当前处理到第 3 个 ASIN" |
| 轻量记录 | Memos API | "安全审计通过" |
| 业务数据 | MySQL | 广告指标、销量数据 |
| 知识文档 | 向量库 | 技能文档、分析报告 |

## 记忆的压缩

### 心跳压缩 (AutoLLMSE-DL)

- **频率**: 每小时
- **目标**: MEMORY.md + memory/*.md
- **方法**: 去除冗余、保留核心信息
- **触发**: 心跳自动执行

### 检查点压缩

- **触发**: 上下文 > 80% / 模型切换前 / 任务交接时
- **动作**:
  1. 更新 `HOT_MEMORY.md`
  2. 追加摘要到 `memory/YYYY-MM-DD.md`
  3. RAG 同步向量化

### 记忆蒸馏 (每周)

- **时间**: 每周日
- **动作**:
  1. 回顾本周 `memory/*.md`
  2. 提取长期有价值的内容 → `MEMORY.md`
  3. 清理过时信息

## 记忆的删除

| 类型 | 策略 |
|------|------|
| 临时文件 | 7 天自动清理 (`/tmp/openclaw_*`) |
| 每日记忆 | 保留，压缩后长期存储 |
| 向量数据 | 不删除，仅追加 |
| 备份文件 | 保留 30 天 |

## 记忆的质量控制

### 写入前检查

1. **唯一性**: 是否已有类似记录?
2. **重要性**: 是否值得长期记住?
3. **结构化**: 是否易于检索?

### 定期审计

- 每月检查 `MEMORY.md` 信息密度
- 清理重复、过时、低价值内容
- 确保关键决策有数据支撑

## 记忆层级迁移

```
会话上下文 ──重要──→ 每日记忆
每日记忆 ──有价值──→ MEMORY.md
MEMORY.md ──知识化──→ 向量库
业务数据 ──分析后──→ MySQL
```

每个迁移方向都意味着信息的进一步提炼和结构化。
